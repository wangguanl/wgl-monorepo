# 06 - 部署与 CI

## 部署目标选型

| 方案 | 说明 | 推荐 |
|------|------|------|
| **A. monorepo GitHub Pages** | 在 `wgl-monorepo` 仓库 Settings → Pages 发布 `docs/.vitepress/dist` | ✅ **默认推荐** |
| B. 继续独立仓库 | 构建后 push 到 `page__package-docs` 等新仓库 | 备选 |
| C. 自定义域名 | `docs.example.com` CNAME 指向 GitHub Pages | 可选，后期 |

### 方案 A 配置要点

- Repository → Settings → Pages → Source: **GitHub Actions**
- Workflow 使用 `peaceiris/actions-gh-pages` 或官方 `actions/upload-pages-artifact`
- VitePress `base` 设为 **`/wgl-monorepo/`**（与 `https://<user>.github.io/<repo>/` 一致）

若将来启用自定义域名，将 `base` 改为 `'/'` 并配置 CNAME。

## 统一部署脚本草案

`scripts/deploy-docs.sh`：

```bash
#!/usr/bin/env bash
set -euo pipefail

pnpm docs:build
# 若用 peaceiris/action，由 CI 上传 artifact，本地脚本仅 build
echo "Built to docs/.vitepress/dist"
```

## GitHub Actions 草案

`.github/workflows/docs.yml`：

```yaml
name: Docs

on:
  push:
    branches: [main]
    paths:
      - 'docs/**'
      - 'packages/**'
      - 'package.json'
      - 'pnpm-lock.yaml'
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - run: pnpm build
      - run: pnpm docs:build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: docs/.vitepress/dist

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```

PR 校验可在现有 `ci.yml` 增加：

```yaml
  docs:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - run: pnpm build
      - run: pnpm docs:build
```

## 旧 URL 兼容

当前对外 URL：

- https://wangguanl.github.io/page__package-utils/
- https://wangguanl.github.io/page__package-plugins/

### 策略 1：旧仓库保留 redirect 页（推荐）

在 `page__package-utils`、`page__package-plugins` 的 `gh-pages` 分支放置最小 HTML：

```html
<meta http-equiv="refresh" content="0; url=https://wangguanl.github.io/wgl-monorepo/packages/utils/" />
```

各用例页可同样 redirect 到新路径，或仅首页 redirect。

### 策略 2：npm homepage 直接改新 URL

各包 `package.json`：

```json
"homepage": "https://wangguanl.github.io/wgl-monorepo/packages/utils/"
```

旧 URL 无 redirect，依赖 npm 与新文档推广。

## turbo 缓存

`docs:build` 声明输出 `docs/.vitepress/dist/**`，依赖 `^build` 确保 packages 已编译（若文档仍引用 dist）。

本地开发 `pnpm docs` 不经过 turbo（persistent dev server）。

## 与现有 deploy.sh 的关系

迁移完成后 **删除**：

- `packages/utils/docs/deploy.sh`
- `packages/plugins/docs/deploy.sh`

不再 force push 到 `page__package-*` 仓库；若选方案 A，仅 CI 部署 monorepo Pages。
