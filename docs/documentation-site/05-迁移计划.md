# 05 - 迁移计划

## Phase 1 — 根目录脚手架（约 0.5 天）

- [ ] 根 `package.json` 添加 `vitepress`、`fast-glob` 与 `docs` / `docs:build` / `docs:preview` 脚本
- [ ] 创建 `docs/.vitepress/config.js`（nav、sidebar 多组、`srcExclude`）
- [ ] 创建 `docs/index.md` 全站首页
- [ ] 更新 `turbo.json` 的 `docs:build` task
- [ ] 本地 `pnpm docs` 验证空站可启动

## Phase 2 — 迁移 utils / plugins 内容（约 1 天）

- [ ] 复制 `packages/utils/docs/examples/` → `docs/packages/utils/examples/`
- [ ] 复制 `packages/utils/docs/index.md` → `docs/packages/utils/index.md`
- [ ] 复制 `packages/plugins/docs/` 同上
- [ ] 批量更新 `@include` 路径：

  | 旧路径 | 新路径 |
  |--------|--------|
  | `../../src/foo.ts` | `../../../packages/utils/src/foo.ts` |
  | `../../dist/es/foo.mjs` | 删除或改为说明性代码块 |

- [ ] 更新文档内 import 示例：`wgl-utils` → `@wgl-m/utils`，`wgl-plugins` → `@wgl-m/plugins`
- [ ] 调整各页内部 link：`/examples/install` → `/packages/utils/examples/install`
- [ ] `pnpm docs:build` 构建通过，逐页 spot check `@include`

## Phase 3 — 补充 node-utils、css（约 0.5 天）

- [ ] `docs/packages/node-utils/index.md` — 包说明、Node 版本要求
- [ ] `docs/packages/node-utils/install.md` — 安装与基本用法
- [ ] `docs/packages/css/index.md` — 样式包概览
- [ ] `docs/packages/css/usage.md` — `@wgl-m/css`、`reset`、`mixin`、`bem` 引入方式
- [ ] config.js 中为两包配置静态 sidebar

## Phase 4 — 部署与 CI（约 0.5 天）

- [ ] 编写 `scripts/deploy-docs.sh` 或 `.github/workflows/docs.yml`
- [ ] 确定 `base` 与 GitHub Pages 来源（见 [06-部署与CI.md](./06-部署与CI.md)）
- [ ] CI 增加 `docs:build` job（PR 校验，不强制 deploy）
- [ ] 首次部署到 staging URL，人工验收

## Phase 5 — 清理与收尾（约 0.5 天）

- [ ] 删除 `packages/utils/docs`、`packages/plugins/docs`
- [ ] 从 utils、plugins 的 `package.json` 移除 vitepress 与 docs 脚本
- [ ] 更新各包 `homepage` 字段
- [ ] 更新根 `README.md` 快速开始（`pnpm docs` 替代 `docs:utils` / `docs:plugins`）
- [ ] 旧 GitHub Pages 仓库添加重定向 HTML（若保留域名）
- [ ] 在 `CHANGELOG` 或发版说明中注明文档 URL 变更

## 风险与回滚

| 风险 | 缓解 |
|------|------|
| `@include` 路径遗漏导致构建失败 | Phase 2 末尾全量 build + 脚本检查 `../../src` 残留 |
| 旧 bookmark / npm homepage 404 | Phase 5 旧站保留 redirect 至少一个 major 版本 |
| sidebar glob 顺序不稳定 | 对 examples 列表 sort by filename |

回滚：保留 Phase 5 删除前的 git tag；旧 `packages/*/docs` 可从 tag 恢复。
