# 03 - 技术选型

## 文档框架：VitePress 1.x

| 选项 | 结论 | 理由 |
|------|------|------|
| **VitePress** | ✅ 采用 | 现有站点已用，Markdown 与 `@include` 无需更换 |
| Docusaurus | ❌ | 迁移成本高，React 栈与仓库风格不符 |
| Storybook | ❌ | 适合组件库，不适合 utils/plugins 函数文档 |

版本：**1.6.3**（与当前 lockfile 一致，升级可另开 PR）。

## 站点根目录

| 选项 | 结论 |
|------|------|
| 根目录 `docs/` | ✅ **推荐** — 与「monorepo 文档」语义一致，README 已指向此目录 |
| `apps/docs/` | ❌ 本仓库无 apps 层，增加目录层级 |
| 保留 `packages/*/docs` | ❌ 无法解决双站点问题 |

VitePress 约定：内容 Markdown 与 `.vitepress/` 同级，即：

```
docs/
├── .vitepress/config.js
├── index.md
├── packages/utils/...
└── typescript-migration/   ← srcExclude 排除
```

从 `@wgl-m/utils`、`@wgl-m/plugins` 的 `devDependencies` 中 **移除 vitepress**；删除 `packages/utils/docs/package.json`、`packages/plugins/docs/package.json`。

`pnpm-workspace.yaml` 移除 `packages/*/docs` 条目（若 docs 下无独立 package.json 则自然失效）。

## 配置策略

### 单文件 `docs/.vitepress/config.js`

- **`base`**：生产环境 `/wgl-monorepo/`（monorepo GitHub Pages）或自定义域名根路径 `/`
- **`srcExclude`**：`['typescript-migration/**', 'documentation-site/**', 'README.md']`
- **nav**：顶层包切换 + 外链（GitHub、npm）
- **sidebar**：按路由前缀拆分，使用 VitePress [sidebar 多组配置](https://vitepress.dev/reference/default-theme-sidebar#multiple-sidebars)

示例结构：

```js
themeConfig: {
  nav: [
    { text: 'Utils', link: '/packages/utils/' },
    { text: 'Plugins', link: '/packages/plugins/' },
    { text: 'Node Utils', link: '/packages/node-utils/' },
    { text: 'CSS', link: '/packages/css/' },
  ],
  sidebar: {
    '/packages/utils/': [ /* 动态 glob examples */ ],
    '/packages/plugins/': [ /* 同上 */ ],
    '/packages/node-utils/': [ /* 静态 */ ],
    '/packages/css/': [ /* 静态 */ ],
  },
}
```

### 动态 sidebar

保留现有 `fast-glob` 扫描模式，路径改为：

```js
fg.sync(['docs/packages/utils/examples/*.md'], { cwd: repoRoot })
// 或在 config 内相对 docs 目录：
fg.sync(['./packages/utils/examples/*.md'])
```

### `@include` 路径

搬迁后从 `docs/packages/utils/examples/debounce.md` 引用源码：

```markdown
<!-- @include: ../../../packages/utils/src/debounce.ts -->
```

规则：**从 `docs/` 内任意 md 到 `packages/<name>/src` 统一为 `../../../packages/<name>/src/...`**（utils/plugins 层级相同）。

不再 `@include` Webpack 时代产物（`dist/es/*.mjs`）；改为 `@include` 源码或 tsup 输出说明（可选折叠展示 dist 结构）。

## 脚本

根 `package.json`：

```json
{
  "scripts": {
    "docs": "vitepress dev docs",
    "docs:build": "vitepress build docs",
    "docs:preview": "vitepress preview docs"
  }
}
```

`turbo.json` 增加：

```json
"docs:build": {
  "dependsOn": ["^build"],
  "outputs": ["docs/.vitepress/dist/**"]
}
```

各包移除 `docs`、`docs:build` 脚本；根目录保留兼容别名一段时间：

```json
"docs:utils": "pnpm docs",
"docs:plugins": "pnpm docs"
```

（两者均打开统一站点，可在文档中注明 deprecated。）

## 搜索

使用 VitePress 默认 **本地搜索**（`themeConfig.search.provider: 'local'`），无需 Algolia 配置。

## 与 TypeScript 迁移的关系

| 项 | 处理方式 |
|----|----------|
| 文档配置 | 保持 JavaScript |
| `@include` 目标 | `.ts` 源码 |
| 示例 import 路径 | 统一为 `@wgl-m/utils`、`@wgl-m/plugins` |
| 构建依赖 | `docs:build` dependsOn `^build`，确保 dist 存在（若文档仍展示产物） |
