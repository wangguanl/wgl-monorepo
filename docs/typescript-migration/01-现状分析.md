# 01 - 现状分析

## 包概览

| 包 | 源码语言 | 构建工具 | 源文件数 | 是否输出类型 |
|----|----------|----------|----------|--------------|
| `@wgl-m/utils` | JavaScript | Webpack 5 + Babel | 15 个 `.js` | 否 |
| `@wgl-m/plugins` | JavaScript | tsup | 14 个 `.js` | 否 |
| `@wgl-m/node-utils` | JavaScript | 无（直发源码） | 1 个 `.js` | 否 |
| `@wgl-m/css` | SCSS/CSS | Gulp | — | 不适用 |

## 当前构建产物

### utils（Webpack 三格式）

| 产物 | 格式 | 用途 |
|------|------|------|
| `dist/main.mjs` | ESM | 现代 bundler / Browser |
| `dist/main.cjs` | CommonJS | Node require |
| `dist/main.js` | ES5 UMD | 老旧 Browser |
| `dist/es/*.mjs` | ESM 分包 | 按需引入 |

### plugins（tsup 双格式）

| 产物 | 格式 |
|------|------|
| `dist/index.js` | ESM |
| `dist/index.cjs` | CommonJS |
| `dist/*.js` / `dist/*.cjs` | 单插件分包 |

### node-utils

| 产物 | 说明 |
|------|------|
| `index.js` | 源码即产物，无编译 |

## 痛点

1. **无类型提示** — 使用者无法获得 IDE 自动补全
2. **构建工具不统一** — utils 用 Webpack，plugins 用 tsup，维护成本高
3. **无 type-check CI** — 重构容易引入运行时错误
4. **node-utils 直发 JS** — 缺少编译步骤，无法输出 `.d.ts`

## 不需要迁移的部分

| 目录 | 原因 |
|------|------|
| `packages/css/` | 纯样式资源，无 TS 业务逻辑 |
| `packages/*/docs/` | VitePress 文档站，保持 `.md` + 现有配置 |
| `packages/utils/scripts/` | 迁移后删除（Webpack 脚本） |
| `packages/css/gulpfile.js` | 构建脚本，可保留 JS 或后续单独改 |
