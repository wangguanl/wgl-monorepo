# 02 - 目标与原则

## 迁移目标

1. 四个 JS 包中 **3 个业务包**（utils / plugins / node-utils）源码改为 TypeScript
2. 发布到 npm 时附带 **`.d.ts` 类型声明**，使用者获得完整 IDE 支持
3. **统一构建工具为 tsup**，移除 utils 的 Webpack 5 流水线
4. 根目录增加 **`pnpm typecheck`**，CI 中强制类型检查

## 设计原则

| 原则 | 说明 |
|------|------|
| **渐进 strict** | 初期 `strict: false`，迁移完成后再开 `strict: true` |
| **API 不变** | 函数签名、导出名称保持不变，避免不必要的 breaking change |
| **最小 diff** | 不做无关重构（如修正 debounce 注释错误留到迁移后可选处理） |
| **css 不动** | 样式包保持 Gulp 流程 |
| **分包独立** | 各包独立 tsconfig，通过 `extends` 共享根配置 |

## 不做的事

- 不迁移 VitePress 文档站到 TypeScript
- 不改动 `@wgl-m/css` 的构建流程
- 不在本次迁移中重写业务逻辑
- 不强制 utils 继续输出 ES5 UMD（`main.js`），见 [07-Breaking-Changes.md](./07-Breaking-Changes.md)
