# 04 - 分阶段计划

## 总览

| 阶段 | 内容 | 预估工时 | 风险 |
|------|------|----------|------|
| Phase 1 | 基础设施 | 0.5 天 | 低 |
| Phase 2 | node-utils 迁移 | 0.5 天 | 低 |
| Phase 3 | plugins 迁移 | 1 天 | 低 |
| Phase 4 | utils 迁移 | 2 天 | 中 |
| Phase 5 | 文档 & CI | 0.5 天 | 低 |
| **合计** | | **~4.5 天** | |

每阶段完成后：`pnpm build` + `pnpm typecheck` 必须通过，再进入下一阶段。

---

## Phase 1：基础设施

### 任务

- [ ] 根目录添加 `tsconfig.base.json`
- [ ] 根 `package.json` 添加 `typescript` 和 `typecheck` 脚本
- [ ] `turbo.json` 添加 `typecheck` task
- [ ] 创建 `docs/typescript-migration/`（本文档，已完成）

### 产出

```
wgl-monorepo/
├── tsconfig.base.json       ← 新增
├── package.json             ← 改 scripts
└── turbo.json               ← 改 tasks
```

### 不涉及源码改动，零 breaking change。

---

## Phase 2：node-utils 迁移

### 为什么先做

- 仅 1 个文件，最小验证
- 验证 tsup + dts + npm exports 流水线
- 独立包，不影响 utils/plugins

### 任务

- [ ] `index.js` → `src/index.ts`
- [ ] 添加 `tsup.config.ts`
- [ ] 添加 `tsconfig.json`
- [ ] 更新 `package.json`（main/types/exports 指向 dist）
- [ ] 添加 `@types/node` devDep
- [ ] 移除 `build: echo` 占位脚本，改为 `tsup`

### 类型要点

```ts
// 示例
readdirAsync: (path: string) => Promise<string[]>
readFileAsync: (path: string) => Promise<Buffer>
isFile: (path: string) => Promise<boolean>
```

### 版本

建议 `@wgl-m/node-utils@2.1.0`（minor：新增类型声明，无 API 变更）

---

## Phase 3：plugins 迁移

### 任务

- [ ] 14 个 `.js` → `.ts`
- [ ] `tsup.config.js` → `tsup.config.ts`，glob 改为 `src/*.ts`
- [ ] 开启 `dts: true`
- [ ] 添加 `tsconfig.json`（lib: DOM, ES2020）
- [ ] 更新 `package.json` exports 加 types 字段
- [ ] `upload.ts` 中 `axios` 保持 external + optional peerDep

### 类型要点

```ts
// Browser 插件统一依赖 DOM lib
export default (element: HTMLElement, callback: () => void): void
export default (str: string): Promise<void>
export default (canvas?: number): void  // scale
```

### 版本

建议 `@wgl-m/plugins@1.1.0`（minor：新增类型声明）

---

## Phase 4：utils 迁移（最大）

### 任务

- [ ] 15 个 `.js` → `.ts`
- [ ] 删除 `scripts/` 整个 Webpack 目录
- [ ] 删除 `babel.config.js`
- [ ] 新增 `tsup.config.ts`（多 entry，含 `es/*` 分包）
- [ ] 新增 `tsconfig.json`（lib: ES2020，不加 DOM — utils 是 universal）
- [ ] 更新 `package.json` build 脚本和 exports
- [ ] 移除 Webpack / Babel 相关 devDependencies

### 复杂文件类型策略

| 文件 | 策略 |
|------|------|
| `deepCopy.ts` | 泛型 `<T>` 或 `unknown` 递归 |
| `debounce.ts` | 明确定义 fn/delay 参数类型 |
| `formatDate.ts` | 日期参数 `Date \| string \| number` |
| `parseUrl.ts` | 返回类型接口 |
| `validation.ts` | 表单规则对象 interface |
| `verifyVar.ts` | 参数 `unknown` |

### 版本

建议 `@wgl-m/utils@3.0.0`（major：移除 ES5 UMD 产物，exports 路径变更）

---

## Phase 5：文档 & CI

### 任务

- [ ] 更新 VitePress 文档中的 `@include` 路径（`.js` → `.ts`）
- [ ] 文档示例补充 TypeScript 用法
- [ ] `RELEASE.md` 补充 `pnpm typecheck` 说明
- [ ] GitHub Actions 加 PR typecheck（可选）
- [ ] 删除 `docs/typescript-migration/` 中「待确认」标记，或归档

### 可选 CI

```yaml
# .github/workflows/ci.yml
on: [pull_request]
jobs:
  typecheck:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20, cache: pnpm }
      - run: pnpm install --frozen-lockfile
      - run: pnpm typecheck
```

## 发版节奏建议

| 阶段完成 | 发版方式 |
|----------|----------|
| Phase 2 | changeset → `@wgl-m/node-utils` minor |
| Phase 3 | changeset → `@wgl-m/plugins` minor |
| Phase 4 | changeset → `@wgl-m/utils` major |
| Phase 5 | 无需发版（仅文档/CI） |

也可全部完成后一次性发版。
